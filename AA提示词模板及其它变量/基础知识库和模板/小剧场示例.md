# Small theater output must be responsive: 
- All outputs must be in pure HTML format(such as `<br>`, `<hr>`, `<b>` or `<p>`).**. Do not use Markdown syntax (such as `**`, `*`, `>` or `---`).   
- Outer container: width 100%, max 1200px, centered, responsive without overflow, box-sizing: border-box.      
- Outer padding: 8–12px on left/right, must not exceed 15px.      
- Inner body text: max width 700–750px, centered, ensuring optimal reading line length.      
- If absolute elements exist, the outer container must be relative.      
- Inside <details>/<div>: width 100%.      
- Body text font size must follow the UI root font (1rem). Minor adjustments allowed between 0.9rem–1rem, not beyond.      
- Background: choose a suitable color or background image consistent with the atmosphere.      
- Decoration: lightweight decorations allowed (e.g., semi-transparent watermark, corner patterns, stamps, gradient borders), must relate to small theater content.      
- Key sentences: automatically pick 2–4 of the most emotionally intense lines/descriptions, beautify them (font size, color, layout, typeface) wrapped in <span> with animation effects.      
- Output must include both the HTML structure and the corresponding <style> animation code.      
- Beautification should be adapted based on the content.      
- Titles must use minimalist Morandi style without background color.      
- If there are multiple modules, generate a small theater for each.
