# Anima 记忆系统核心机制说明文档

本说明文档将 Anima 辅助工具的系统架构分为核心六大模块：总结与记录、向量 RAG、BM25 检索、知识库、状态变量以及相关配置设定。已省略非必要教程指引，专注阐述系统判定逻辑、配置规范及其运作原理。

## 1. 总结与记录模块 (Summary & Record)

**核心功能**：定时自动对当前对话进行总结，并通过切片化手段保存剧情段落。

*   **自动化总结触发逻辑**：
    *   遵循设定的【触发间隔】截断并总结历史楼层。
    *   **完整性保障**：若触发层为底层模型回复层，插件将挂起任务，等待人类 User 发送下一楼层后再进行合并总结，以确保交互连续性。
*   **总结向量化与持久化**：
    *   自动总结或通过界面“历史总结管理”手动修改更新的总结片段，将被系统自动向量化。
    *   总结内容按照发生时间依序写入至【聊天世界书】特定节点（如 `chapter_n`）实现持久化。如果是跨窗继续，老窗口不经 Anima 托管的纯文本总结将无法进行切片检索引擎的调用（除非手动补全向量化）。
*   **切片命理（ID机制）**：总结历史生成的局部片段以 `#N_M` 标识（例：`1_2` 代表序号第 1 批次总结返回的第 2 个切片）。

## 2. 向量 RAG (Vector RAG)

**核心功能**：配合总结生成的切片与标签配置发送动态的记忆回溯。系统将 RAG 切分为基础、复杂类型分类分别触发，防止语义过度塌陷向单一向量域。

*   **数据库类型**：
    *   **聊天数据库**：分为“Current”（本窗）与其他跨窗关联数据库（支持分布式标签提取）。合并旧窗库时，名字严禁以 `kb_` 开头。
    *   **知识库**：通过文档直接生成，必须以 `kb_` 命名且不支持分布式标签。
*   **分布式检索策略机制**：
    1.  **基础检索**：单纯依据最高语义相关度返回 N 个基础切片，记录高频标签。
    2.  **重要检索**：提取被强制关联（如 `Important`）等重置型标注重切片。
    3.  **节日/生理检索**：基于全局**虚拟时间**触发。虚拟时间必需含“年月日”且属性层级深度在 YAML 表单三层内，与预设日期区间匹配时强制抽取。
    4.  **状态检索**：与【状态变量】联动，根据当前的 YAML 即时数值或路径映射规则返回带标签记忆（例：如路径等于 `生病`，则返回 `Sick` 标签记忆）。
    5.  **丰富度检索**：排除在前序步骤被提取过标签相关维度的同质记忆切片后，提供相关度内的剩余切片供给。
*   **增益系统**：
    *   **近因加权 (Recency Weighting)**：加权 Current 数据库相关检索分数分发，削弱过远跨窗库记忆导致的污染。
    *   **记忆回响 (Memory Echo)**：检索抽离的记忆体被附带“粘性 (Life)”。即便该内存在下一次 RAG 时并未受触发要求，系统凭借粘性保留在 Top 余量缓存里平滑递减传送输出，防止“记忆闪烁”。
    *   **结果重排 (Rerank)**：由专有 Reranker 模型对待重选的【基础】与【重要】集进行二次排序返回精排切片（节日/生理/丰富度跳过重拍直出）。

## 3. BM25 检索 (BM25 Keyword Retrieval)

**核心功能**：使用本地轻量化检索引擎取代/增强常态全局关键词“绿灯”。
*   **索引词与触发词管理**：
    *   索引词（词法原名）与 逗号分割的触发词（例如专属小名）。
    *   V3.0 预设在聊天终点会自动提取并更新当前词典内容并重建检索库树丛。
*   **意图识别**：若当前 User 上下文同时含有触发关键词及类似“第一次/最后一次”描述时，检索引擎截断常规排序，强制置顶关联历史特征最鲜明的初见/终端切片。

## 4. 知识库管理 (Knowledge Base)

**核心功能**：无标补充拓展世界观文档设定。
*   **切分与注入**：支持 TXT 和 Markdown 上传，切分机制可依照指定【自定义切分符】强制切开（会吞并字符数限制）或智能均字切断。
*   产出使用独立 BM25/向量模型混查最终推入由 `{{knowledge}}` 宏引用的前排提示词阵列。

## 5. 状态变量交互 (State Variables Management)

**核心功能**：通过专门的状态小模型跟踪维护一个动态 YAML 对象面板。
*   **YAML 架构更新要求**：
    *   支持状态推导和更新通过 JSON 对象发送。建议使用如 `"HP": "+50"` 的计算表达式让 Anima 直接换算以免除 LLM 智障计算风险。
*   **数据清洗 (State Cleaning)**：
    *   防止随会话发展使得记录爆档或重覆累加，可手动唤出【校准提示词】借用总结级主模型去二次精简 YAML 内容树（增/修/删）。
*   **数据强校验 (Output Validation)**：
    *   非脚本模式：强制约束数值区间（Min/Max/振幅）、选项枚举类型限制（未枚举则转 String），或强转布尔值。
    *   脚本模式 (Zod)：可在后台返回内加入脚本进行回调操作（例如结合自带的 `utils.autoNum` 可以实现最大差额 Delta 约束或 fallback 退守还原记录）。
*   **动态宏 `_user` 和 `_char`**：在使用多卡片人物和开场设定时，可用前置下划线引用通用对象的变量（例如：变量校验路径书写为 `_user.HP`）。

## 6. 其他配置规范与美化
*   **提示词过滤清洗**：系统允许使用正则表达式拦截思维链标签。处理前文或提交给 RAG 时建议剔除思维流向。
*   **特定占位符声明**：
    *   `{{chatHistory}}`：注入聊天记录（向量 RAG + BM25 的组合处理内容）—— 深度统一恒插在前排 (Depth: 9999)。旧版占位为 `{{rag}}` 功能相等。
    *   `{{ANIMA_BASE_STATUS}}`：全局暴露在世界表任何区域抛出目前存储的状态 YAML 代码。附带子级检索。
*   **状态栏 UI 渲染逻辑**：
    *   采用原生的 Handlebars 模板处理：`{{#each 目标路径}}` 可用于遍历物品栏或对象子级变化，`{{@key}}` 映射对象子级动态名称。
    *   `{{#if 目标 >= 常量}}`：可引入与 (`&&`)、或 (`||`) 等简单比较拦截，杜绝 if 内部层进循环嵌套。
    *   操作逆向回调支持在 onclick 事件通过原生 JS `_.set` 返回更改状态池中的变量，发起广播实现真正的动态化界面。
