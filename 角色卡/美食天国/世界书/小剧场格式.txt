# Small Theater Output Structure and Style Guide

## 1. Basic HTML Structure
You must strictly follow the HTML structure below for wrapping small theater content:
<section>
    <details>
        <summary style="list-style:none; text-align:center;">{{Dynamic Title}}</summary>
        <div style="white-space: pre-line;">
            <!-- 此处为小剧场的具体HTML内容 -->
            {{SmallTheaterContent}}
        </div>
    </details>
</section>

## 2. Responsive Layout and Readability Requirements
- Pure HTML Format: All output content must be pure HTML tags (e.g., `<p>`, `<div>`, `<span>`, `<b>`, `<hr>`). Markdown syntax (e.g., `**`, `---`) is strictly forbidden.
- Outer Container: The container width should be 100%, with a maximum width not exceeding 1200px, centered, ensuring no overflow on any device.
- Padding: The left and right sides of the container should have padding between 8px and 12px, not exceeding 15px.
- Body Text Area: The maximum width of the core text content should be limited to 700-750px and centered, to ensure optimal reading line length.
- Font Size: Body text font size should generally follow the UI's root font size (1rem), with minor adjustments allowed between 0.9rem and 1rem.

## 3. Visual Style
- Background and Color: choose a suitable color or background image consistent with the atmosphere.
- Decoration: lightweight decorations allowed (e.g., semi-transparent watermark, corner patterns, stamps, gradient borders), must relate to small theater content.
- Title: The title within the `summary` tag should maintain a minimalist style.
- Output must include both the HTML structure and the corresponding <style> animation code.      
- Beautification should be adapted based on the content.
