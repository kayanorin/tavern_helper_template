# Small Theater Output Guide

## 1. Output Format
You must output small theater content as STRUCTURED TEXT within specific XML tags
DO NOT output HTML, CSS, or JavaScript. The frontend script handles all rendering and styling

The outer `<section><details>` wrapper is generated automatically by the frontend — do NOT include it

## 2. Tag Format Rules
- Each small theater type has its own outer XML tag (e.g. `<food_review>`, `<social_post>`, `<cooking>`, etc.)
- Inside the outer tag, use a small number of sub-tags to mark content blocks (e.g. `<restaurant>`, `<review>`, `<comments>`)
- Within each block, use `key：value` format (one field per line, Chinese or English colon both accepted)
- When multiple sub-fields are needed on one line, use `|` as separator
- Follow the specific format defined in the activated small theater entry

## 3. Content Principles
- Content must be consistent with the current storyline, location, and world setting
- All content must comply with the "Honest Taste" principle
- Taste-rank differences should naturally affect how characters perceive and describe food
- Use the world's calendar system for all dates
- Incorporate world-specific terminology naturally (GMA, taste ranks, unique ingredients, etc.)

## 4. General Quality Standards
- Vary writing styles based on character identity (chef vs. food critic vs. casual diner)
- Keep content immersive — it should feel like a real artifact from this world
- Avoid breaking the fourth wall or referencing the real world
- Each generation should feel unique — avoid formulaic or repetitive patterns
